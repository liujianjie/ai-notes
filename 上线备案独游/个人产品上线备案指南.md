# 个人产品上线备案指南（微信小程序 / 网站 / 安卓）

> 面向 `衡仓`（资产配置追踪器）这类个人开发者项目。涵盖网站 ICP 备案、APP 备案、微信小程序主体注册、是否需要注册工作室 / 公司、完整成本估算。
>
> ⚠️ **免责声明**：以下内容基于截至 2026-05 的公开监管政策整理，实际要求可能随工信部 / 网信办 / 证监会政策变动。下单前务必查询官方入口（[beian.miit.gov.cn](https://beian.miit.gov.cn)、微信公众平台）确认最新规则。

---

## TL;DR — 决策树

```
产品面向谁？
├─ 只给自己 / 家人 / 朋友私下用
│   └─ ✅ 境外托管（Cloudflare Pages / Vercel）+ 不备案 + 安卓 APK 手动分发
│       成本：仅域名（~$10/年），无备案，无主体要求
│
└─ 开放给陌生用户 / 上架应用市场 / 上线小程序
    ├─ 产品是否涉及股票/证券/投资推荐等"金融类"？
    │   ├─ 是 → ⚠️ 个人主体基本走不通，强烈建议至少注册个体工商户
    │   │   （衡仓涉及持仓数据、配比建议，易被认定为金融类）
    │   └─ 否（纯工具/效率类）→ 个人主体大概率可行
    │
    └─ 是否需要部署在国内服务器？
        ├─ 是 → 必须 ICP 备案 + 公安备案
        └─ 否（境外托管）→ 可免 ICP 备案，但安卓 App / 微信小程序仍需备案 / 主体审核
```

## 一、关键概念速览

| 术语 | 说明 | 个人能做吗 |
|------|------|----------|
| **域名实名认证** | 所有域名注册后必须做，上传身份证。免费 | ✅ 可 |
| **ICP 备案** | 网站部署在**境内服务器**时必须。工信部统一管理，免费 | ✅ 可（限制多） |
| **公安备案** | ICP 备案通过后 30 天内必须完成。免费 | ✅ 可 |
| **ICP 许可证（经营性）** | 收费 / 有交易功能的网站需要。需要企业主体 | ❌ 不可 |
| **APP 备案** | 2023-09 起工信部要求，2024-03-31 起全面执行。免费 | ✅ 可 |
| **微信小程序主体认证** | 个人免费（限制多），企业 300 元/年 | ✅ / ✅（分类型） |
| **营业执照** | 企业主体的凭证。个体工商户 or 公司 | 需主动办理 |

## 二、境外托管 vs 境内托管 — 最大分叉点

### 境外托管（推荐给个人项目）

**托管在**：Cloudflare Pages / Vercel / Netlify / GitHub Pages / 海外 VPS

- ✅ **免 ICP 备案**（服务器不在境内）
- ✅ 免费额度足够个人项目用
- ✅ 部署快，无审核等待
- ⚠️ 国内访问速度中等（Cloudflare 国内路线尚可；Vercel / Netlify 较慢，可能被偶发封锁）
- ⚠️ 不能用阿里云 / 腾讯云 CDN 加速（CDN 节点要求域名备案）
- ⚠️ 微信内打开 H5 可能被风控拦截（"即将离开微信"弹窗）

### 境内托管

**托管在**：阿里云 / 腾讯云 / 华为云等境内服务器或 CDN

- ❌ 必须 ICP 备案 + 公安备案
- ❌ 备案周期 7–20 天，期间网站必须关闭
- ✅ 国内访问速度快
- ✅ 可用阿里云 / 腾讯云 CDN、对象存储直链
- ✅ 可申请微信小程序 `webview` 白名单、支付等能力

**衡仓建议**：先走境外托管（Cloudflare Pages），不备案。等用户量起来、确定要做微信小程序支付 / 接入境内 CDN 时再考虑备案。

## 三、网站 ICP 备案详解（仅境内托管需要）

### 个人备案流程

1. **买云服务器/OSS**：阿里云 / 腾讯云国内节点，**至少 3 个月以上订单**才能用于备案
2. **登录云厂商的备案系统**：如阿里云 `beian.aliyun.com`
3. **提交材料**：
   - 身份证正反面照片
   - 手持身份证照片
   - 手机号（备案手机需与实名一致）
   - 网站负责人信息、网站名称、网站用途
   - 域名证书（云厂商后台下载）
4. **幕布拍照**：云厂商寄幕布（纯色背景），站到幕布前拍合规照
5. **管局审核**：提交工信部管局审核，**7–20 工作日**
6. **备案号**：通过后拿到 `京ICP备XXXXXXXX号-1`，必须放到网站页脚

### 个人备案的内容红线

个人备案**不允许**出现：

- ❌ "企业""公司""商城""平台"等字样
- ❌ 交易 / 支付 / 购物功能
- ❌ 论坛 / BBS / UGC 互动内容
- ❌ **股票推荐 / 投资建议 / 金融资讯** ← 衡仓重点避雷
- ❌ 新闻 / 出版 / 医疗 / 教育培训内容

如果衡仓的网站含"AI 股票咨询""涨跌预测"等功能，**个人备案极可能被驳回**，且备案号审查是长期的（一年一复查），被发现会注销备案。

### 公安备案

- ICP 备案通过后 30 天内必须做
- 入口：[www.beian.gov.cn](https://www.beian.gov.cn/)
- 材料比 ICP 少，审核 3–7 天
- 通过后拿到公网安备号，同样放页脚

## 四、安卓 App 备案详解

### 政策背景

- **2023-08 工信部发文**：所有在国内运营的 App 必须完成备案
- **2024-03-31 起**：未备案 App **全面下架**，云服务 / CDN / 推送等基础服务对未备案 App 断供
- 覆盖：应用商店上架 + 官网下载 + 二维码分发

### 个人备案流程

1. 类似 ICP 备案，走云厂商通道（阿里云 / 腾讯云）
2. 材料：
   - 身份证
   - **App 域名**（后端 API 用的域名，需先完成 ICP 备案）
   - App 签名证书 SHA-1（用 `keytool` 或 `apksigner` 生成）
   - App 功能说明、包名
3. 审核周期 7–20 天
4. 备案号需展示在 App"关于页"

### 特殊风险：金融类 App

工信部对"金融类"App 备案**额外要求**：

- 需提供《金融许可证》《证券投资咨询业务资格证书》等
- **个人主体基本无法通过**金融类审核
- 即使申报为"工具类"，一旦功能涉及股票数据展示 / 推荐，可能被应用商店下架

**衡仓规避策略**：

- 如果只自用：**不上架、不备案**，手动安装 APK（Android 允许"未知来源"安装）
- 如果要分发给其他人：
  - 功能上严格去掉"推荐""建议""预测"字样
  - 定位为"个人持仓记录工具"
  - 仍建议至少有个体工商户执照

## 五、微信小程序详解

### 主体类型对比

| 主体类型 | 认证费 | 可选类目 | 衡仓能用吗 |
|---------|-------|---------|----------|
| **个人** | 免费 | 工具、教育（部分）、出行、生活服务（部分） | ⚠️ 仅限纯工具功能，不能有"金融"字样 |
| **个体工商户** | 300 元/年 | 比个人多出商业、电商等类目 | ✅ 可以申报工具类 |
| **企业** | 300 元/年 | 全类目，含部分金融（仍需资质） | ✅ 首选 |
| **政府 / 媒体 / 其他组织** | 300 元/年 | 特殊类目 | — |

### 金融类类目的致命限制

微信小程序**严格管控金融类目**，包括：

- 银行 → 需要银保监会许可证
- 证券 / 期货 / 基金 → 需要证监会许可证
- **股票 / 证券行情、股票推荐** → 必须是证监会批准的证券公司 / 投资咨询机构
- 投资理财 / 借贷 → 金融牌照

👉 **衡仓如果功能写"股票配置追踪""涨跌分析"，大概率被拒**。

### 可行的变通方案

申报类目选 **"工具 → 效率"** 或 **"工具 → 记账"**，功能描述为：

> "个人资产记录与配比可视化工具，帮助用户手动录入自己的持仓数据并查看占比分布。不提供任何投资建议、股票推荐、收益预测。"

需要同步调整：

- 小程序名 **不含**"股""证券""投资""理财"等敏感词（"衡仓"无敏感词 ✅）
- 介绍文案淡化"股票"字样，强调"资产""记账""工具"
- **禁用** AI 股票咨询 / 涨跌预测功能（即使后端实现，小程序前端不提供入口）

### 注册流程

1. [mp.weixin.qq.com](https://mp.weixin.qq.com) → 注册 → 选主体类型
2. 个人：上传身份证 + 手持照，微信扫码人脸核身
3. 企业 / 个体工商户：上传营业执照 + 法人身份证 + 对公打款验证（几毛钱）或微信认证（300 元）
4. 小程序名称审核 1–7 天
5. 提交代码审核前需完成：**类目申报**、**服务器域名白名单配置**（需 ICP 备案的域名）、**隐私政策提交**
6. 代码审核 1–7 天
7. 通过后发布

## 六、主体类型：个人 vs 个体工商户 vs 有限公司

### 成本对比

| 维度 | 个人 | 个体工商户（工作室） | 有限公司 |
|------|------|-------------------|---------|
| **注册费** | 0 | 200–500 元（代办）/ 自办免费 | 0–2000 元（各地政策差异大） |
| **年度成本** | 0 | 0（不刻章不代账可以）–1200 元（代账） | 2400–6000 元（代账 + 年报 + 社保） |
| **税务** | 无 | 按季度申报，多数省份月销 10 万以下免增值税 | 月季年均需报税，规范度要求高 |
| **银行账户** | 个人储蓄卡 | 可开对公（非必须） | **必须开对公户**，年费 300–1200 元 |
| **小程序类目** | 最少 | 中等 | 最多 |
| **域名备案** | 个人备案（限制多） | 企业备案 | 企业备案 |
| **退出成本** | 0 | 简易注销免费 | 正常注销 1000–3000 元，涉及税务清算 |

### 什么时候值得注册个体工商户

- 要上线**商业化**微信小程序（开通支付、企业主体认证）
- 要做 **金融 / 教育 / 医疗 / 出版**等敏感类目
- 要开**发票**（个人主体无法开增值税发票）
- 要接广告联盟 / 对公收款
- 要在**应用市场**上架（小米 / 华为应用市场要求开发者为企业）

### 个体工商户注册简流程

1. **起名核准**：本地"企业开办一网通办"平台（各省名字不同，如北京"e 窗通"、上海"一网通办"）
2. **提交材料**：身份证、经营场所证明（住宅地址多数省份允许，需房产证 + 无偿使用证明）
3. **领取营业执照**：3–7 工作日，部分地区当天可取
4. **刻章**：公章 + 财务章（约 200 元，部分地区免费送）
5. **开户**：带营业执照去银行开对公户（非必须，个人账户也能收款）
6. **税务登记**：电子税务局激活，按季度零申报

**名字建议**：`XXX工作室` / `XXX科技工作室` / `XXX信息咨询工作室`，经营范围写"计算机软件开发、信息技术咨询、数据处理"等。

### 有限公司 vs 个体工商户

| 需求 | 推荐 |
|------|------|
| 个人项目验证期，每年收入 < 10 万 | 个体工商户 ✅ |
| 要拿风投 / 股权融资 / 发股份 | 有限公司 |
| 承担责任（债务 / 合同纠纷）有限 | 有限公司（个体工商户无限责任） |
| 要申请软著 / 专利 | 都可以，企业更方便 |

**衡仓建议**：个体工商户足矣，成本低、注销快、适配小程序 / App 备案。

- ⚠️ 金融类内容风险高，小程序功能受限

- 域名：70 元 / 年
- 境内云服务器：300–800 元 / 年（或境外免费 + 不备案）
- ICP 备案：免费
- APP 备案：免费
- 微信小程序认证：300 元 / 年
- 代账（可选）：0–1200 元 / 年
- **首年合计：~1000–3000 元**
- **次年起：~700–2400 元 / 年**

### 路径 E：有限公司（不推荐给个人项目）

- 首年 ~5000–10000 元
- 次年起 ~3000–6000 元 / 年
- 仅建议融资 / 多人合伙 / 规模化后考虑

## 八、针对衡仓的具体建议

### 阶段 1：MVP 自用期（当下）

- ✅ 境外托管（Cloudflare Pages）
- ✅ 仅自己 + 朋友用，不备案
- ✅ 安卓出 APK，自己装
- ✅ 小程序**暂缓**，等主体问题想清楚

### 阶段 2：小范围推广期（有 10–100 用户时）

- 保持境外托管
- 安卓仍走 APK（或发布到 GitHub Release）
- 考虑注册**个体工商户**（`XXX 科技工作室`），~500 元一次性，为小程序铺路
- 微信小程序以"工具 / 效率"类目注册，严格避开金融字样

### 阶段 3：正式上线（有支付 / 商业化需求时）

- 个体工商户 → 视需要升级为有限公司
- 网站做企业 ICP 备案（境内加速）
- App 备案
- 小程序企业认证 + 开通支付

### 红线（任何阶段都不要踩）

- ❌ 在小程序 / 备案网站公开"股票推荐""涨跌预测""买入建议"
- ❌ 用未实名的他人域名 / 主体做备案
- ❌ 备案号 / App 备案号不展示在规定位置
- ❌ 擅自更换备案信息后的服务器 IP 不报备
- ❌ 金融内容用 AI 生成 + 公开分发（涉及证券法 / 广告法双重风险）

## 九、常见疑问

**Q：我用境外服务器 + 国外域名（.com / .app），是不是就完全不用备案？**
A：网站层面确实不用 ICP 备案。但：(1) 安卓 App 只要国内用户能装、用到国内 API 域名，2024 起就要 App 备案；(2) 微信小程序即使后端部署海外，主体注册和类目审核仍然必须；(3) 应用市场上架要求备案号。

**Q：个体工商户和个人，对外感觉区别大吗？**
A：对用户几乎无感。但对微信 / 应用市场 / 银行，差别很大。个体工商户能开对公户、开发票、认证企业小程序、上应用市场，个人基本都做不了。

**Q：工作室和公司税负差多少？**
A：个体工商户年营业额 10 万以下多数省份免增值税，所得税按经营所得 5%–35% 累进，规模小的情况下税负低、账目简单。有限公司增值税 6% 起、企业所得税 25%（小微减半 5%），叠加代账费更高，适合规模化后用。

**Q：没有实体办公场所能注册工作室吗？**
A：大部分省份允许住宅地址（需房产证 + "住改商"证明或社区盖章）。也有地区提供"集群注册"地址（园区代办，~500–2000 元 / 年）。**深圳 / 北京 / 上海**部分区限制较严，**成都 / 杭州 / 武汉**等二线城市更宽松。

**Q：注册工作室需要本人到场吗？**
A：现在多数省份全流程线上办理 + 人脸核身，不用跑工商局。银行对公户开户**必须**法人到场一次。

## 十一、自用期的免费 / 低成本托管方案（衡仓实战）

衡仓的技术栈是 **FastAPI（Python 后端）+ SQLite + Vue 3 PWA / uniapp**。前端纯静态部署哪里都免费，**难点在后端**——需要常驻进程、持久化数据、偶尔跑 OCR / 调 Anthropic API。

### 架构分层视角

```
┌──────────────────────────────────────────────┐
│ 前端（PWA / H5 / 小程序 / Android）            │  → 哪里都能部署，天然免费
├──────────────────────────────────────────────┤
│ 后端 API（FastAPI）                           │  → 难点：需 24/7 在线 + 持久化
├──────────────────────────────────────────────┤
│ 数据存储（SQLite 文件 + OCR 截图）             │  → 难点：免费方案大多没持久化磁盘
└──────────────────────────────────────────────┘
```

### 自用期 4 种免费方案

| 方案 | 后端跑在 | 外网访问 | 数据持久化 | 可靠性 | 推荐指数 |
|------|---------|---------|----------|--------|---------|
| **A. Cloudflare Tunnel + 家里电脑** | 自己的 PC / NAS / 树莓派 | Cloudflare Tunnel（免费） | 本地磁盘，无限 | 取决于你电脑 24h 开不开 | ⭐⭐⭐⭐⭐（真·零成本） |
| **B. Fly.io** | Fly.io 容器 | 自带 HTTPS + 域名 | 3GB 免费持久卷 | 高（有 free credit $5/月） | ⭐⭐⭐⭐ |
| **C. Oracle Cloud Always Free** | 甲骨文 ARM VM | 自带公网 IP | 永久 200GB | 高但账号难申请 | ⭐⭐⭐（门槛高） |
| **D. Render / Railway** | 免费 Web Service | 自带 HTTPS | ❌ 无持久卷（SQLite 会丢） | 低（Render 15min 不用就 sleep） | ⭐⭐（不推荐 backend） |

### 方案 A：Cloudflare Tunnel + 家里电脑（最推荐）

**原理**：家里电脑跑 FastAPI（`127.0.0.1:8000`），装 Cloudflare 官方客户端 `cloudflared`，它主动向 Cloudflare 建立出站隧道。用户访问 `api.hengcang.com` → Cloudflare 边缘 → Tunnel → 你家电脑。

**优点**：

- ✅ 完全免费（Cloudflare Tunnel 免费无流量限制）
- ✅ **不需要公网 IP、不需要端口转发、不需要改路由器**（出站连接即可）
- ✅ 自动 HTTPS，自带 DDoS 防护
- ✅ 数据 100% 在自己手里，SQLite 文件就在本地
- ✅ 后续"丝滑转换"时，Tunnel 配置可以直接指向云 VPS，**零迁移成本**

**缺点**：

- ⚠️ 电脑关机 / 断网 / 宽带重启就下线
- ⚠️ 家用上行带宽一般只有 30–100 Mbps（但对衡仓单用户够了）
- ⚠️ 如果在微信里打开 H5，偶尔会触发"即将离开微信"提示

**部署步骤**：

```bash
# 1. 装 cloudflared（Windows）
# 下载：https://github.com/cloudflare/cloudflared/releases

# 2. 登录 Cloudflare（浏览器弹窗授权你的域名）
cloudflared tunnel login

# 3. 创建隧道
cloudflared tunnel create hengcang-home

# 4. 配置路由（把 api.hengcang.com 指向本地 8000 端口）
cloudflared tunnel route dns hengcang-home api.hengcang.com

# 5. 写配置文件 ~/.cloudflared/config.yml
tunnel: hengcang-home
credentials-file: C:\Users\<你>\.cloudflared\<tunnel-id>.json
ingress:
  - hostname: api.hengcang.com
    service: http://localhost:8000
  - service: http_status:404

# 6. 启动（或做成 Windows 服务开机自启）
cloudflared tunnel run hengcang-home
```

**配套建议**：家里电脑装 Windows 自动登录 + 设置不休眠 + 用 NSSM 把 FastAPI 和 cloudflared 都做成 Windows 服务。更稳的话可以花 500–1500 元配个小主机或买二手 ThinkCentre / 群晖 NAS 常年挂机。

### 方案 B：Fly.io（不想自己挂机的最佳选择）

**原理**：Fly.io 是容器 PaaS，全球边缘节点，Dockerfile 一键部署。

**额度**（2026 年最新）：

- 每月 $5 免费 credit，够小项目跑 1 个共享 CPU + 256MB RAM 实例全月
- 3GB 持久卷免费（SQLite 数据落这里）
- 自带域名 `hengcang.fly.dev` + 免费 SSL，可绑定自有域名

**部署步骤**：

```bash
# 1. 装 flyctl
# https://fly.io/docs/hands-on/install-flyctl/

# 2. 登录
flyctl auth login

# 3. 项目根目录写 Dockerfile（见下一节）

# 4. 初始化（自动生成 fly.toml）
flyctl launch

# 5. 创建持久卷（存 SQLite + OCR 截图）
flyctl volumes create hengcang_data --size 3

# 6. 部署
flyctl deploy
```

**缺点**：国内访问偶尔会慢 / 不稳定（香港节点相对较好），长期看仍建议过渡到国内 VPS。

### 方案 C：Oracle Cloud Always Free

**额度**：

- 4 核 ARM CPU / 24GB RAM 可自由分配（牛逼到不合理）
- 200GB 块存储
- 10TB / 月出站流量
- **永久免费**

**门槛**：

- 信用卡验证严格，**国内信用卡 / 双币卡通过率低**，需 Visa / Mastercard
- 注册时要选"Always Free 区域"，国内离日本 / 韩国区节点近
- 长时间不登录 / 闲置可能被回收（2024 后政策摇摆过）

**适合**：有海外信用卡、折腾能力强的人。配出来性能直接顶国内 50 元/月的轻量云。

### 方案 D：Render / Railway（不推荐用于衡仓后端）

- **Render Free**：服务 15 分钟无请求就 sleep，下次请求冷启动 30 秒+。没有持久卷 = SQLite 每次重启清零。**不适合 backend**。
- **Railway**：2023 起取消真·免费层，现在是 $5 trial credit 一次性，耗完就收费。

**唯一适合这俩的场景**：纯前端静态站 / 偶发触发的 webhook，不是衡仓后端。

## 十二、"丝滑转换"的架构要点（自用 → 公开）

"丝滑"的关键是**基础设施可替换、代码零改动**。需要在自用期就做好以下 4 件事：

### 1. 后端容器化（写 Dockerfile）

衡仓后端示例 `backend/Dockerfile`：

```dockerfile
FROM python:3.12-slim

WORKDIR /app

# 系统依赖（PaddleOCR 需要）
RUN apt-get update && apt-get install -y --no-install-recommends \
    libgl1 libglib2.0-0 && \
    rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

# 持久化目录挂载点
VOLUME ["/app/data"]

ENV PORT=8000
EXPOSE 8000

CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

有了这个 Dockerfile，下面任何平台都能部署（Fly.io / Railway / 阿里云容器实例 / 自建 VPS），**代码一行不用改**。

### 2. 配置外置（12-Factor）

**所有环境相关的值走环境变量**，绝不硬编码：

```python
# backend/config.py
import os
from pathlib import Path

# ❌ 不要这样
# DB_PATH = "/mnt/c/Users/Administer/Desktop/hengcang.db"

# ✅ 应该这样
DB_PATH = Path(os.getenv("HENGCANG_DB_PATH", "./data/hengcang.db"))
ANTHROPIC_API_KEY = os.getenv("ANTHROPIC_API_KEY")  # 绝不写默认值
OCR_UPLOAD_DIR = Path(os.getenv("HENGCANG_UPLOAD_DIR", "./data/uploads"))
CORS_ORIGINS = os.getenv("HENGCANG_CORS_ORIGINS", "http://localhost:5173").split(",")
```

本地用 `.env` 文件（加 `.gitignore`），线上用各平台的 Secrets / 环境变量面板。这条和 [CLAUDE.md 铁律 2] 一致。

### 3. 数据库：SQLite → PostgreSQL 的迁移准备

衡仓的 [`开发列表.md`](F:\AIProject\StockTrading\提示词\DevNote\开发列表.md) 里已经写了"数据库要切换"。**一开始就用 SQLAlchemy（或其他 ORM）**，这样未来换 Postgres 只需要改 `DATABASE_URL`：

```python
# 自用期
DATABASE_URL=sqlite:///./data/hengcang.db

# 开放期
DATABASE_URL=postgresql://user:pass@host:5432/hengcang
```

**Alembic 迁移脚本从第一天就建立**，切换数据库时自动应用 schema。

**迁移工具**：`pgloader`（SQLite → Postgres 一键迁移，保留数据）。

### 4. 文件存储抽象（OCR 截图 / 用户上传）

自用期文件存本地：`./data/uploads/`。开放期换成对象存储（多用户时本地磁盘扛不住）。

**推荐用 S3 兼容接口封装一层**：

```python
# backend/storage.py
class Storage:
    def save(self, key: str, content: bytes) -> str: ...
    def get(self, key: str) -> bytes: ...

class LocalStorage(Storage): ...       # 自用期
class S3Storage(Storage): ...          # 开放期（Cloudflare R2 / 阿里云 OSS）

# 根据 env 切换
storage = S3Storage() if os.getenv("S3_ENDPOINT") else LocalStorage()
```

**对象存储首选**：**Cloudflare R2**（免费额度 10GB 存储 + 零出口流量费，和 Cloudflare Pages / Tunnel 同生态）。

## 十六、量化交易工具与衡仓并行运行指南

假设你还有一个**量化交易工具**需要 24/7 实时跑（盯盘 / 策略信号 / 自动下单），这个场景与衡仓的需求**完全不同**，必须分开设计，否则量化挂了、单也下错了、钱可能是真亏。

### 两类工具的核心差异

| 维度 | 衡仓（Web 工具） | 量化交易工具 |
|------|----------------|------------|
| **在线要求** | 用户用时能打开即可 | 交易时段**一秒不能断**（A 股 9:25–15:00） |
| **延迟敏感度** | 秒级可接受 | 毫秒级（日内策略）/ 秒级（日频策略） |
| **故障代价** | 用户刷新 / 数据稍晚 | **真金白银亏损**，可能单边错仓 |
| **网络需求** | 普通家宽足够 | 需稳定低延迟 + 主备线路 |
| **对外暴露** | API 要让用户访问 | **绝不对外暴露**，只对券商 API 出站 |
| **密钥敏感度** | Anthropic API Key | 券商账密 + 资金密码（风险等级最高） |
| **资源占用** | 低（除非 OCR） | 中高（行情订阅 / 因子计算） |

**关键结论**：量化和衡仓**不能混在一个进程 / 一个容器**里跑，最好连机器都分开。

---

### 三种共存架构

| 方案 | 量化跑在 | 衡仓跑在 | 月成本 | 适合 |
|------|---------|---------|--------|------|
| **A. 单台主机 + Docker 隔离** | 本地台式机 | 同机 Docker | 电费 ~30 元 | 小白 / 预算敏感 |
| **B. 双机分离**（⭐推荐） | 专用台式机 / 工控机 | 旧电脑 / 小主机 | 电费 ~50 元 | 有真金投入的量化 |
| **C. 券商托管 VPS + 家里衡仓** | 券商云托管 | 家里电脑 + Tunnel | 100–500 元/月 | 低延迟策略 / 出差多 |

### 方案 A：单机 Docker 隔离（最经济）

**架构**：

```
┌──────────────────────────────────────────────────────┐
│ 一台台式机（推荐 i5+ / 16GB+ / SSD / 千兆网卡）        │
│                                                       │
│  ┌─────────────────┐      ┌─────────────────────┐   │
│  │ 量化容器         │      │ 衡仓容器             │   │
│  │ vnpy / 自研策略  │      │ FastAPI + SQLite    │   │
│  │ CPU限额: 70%    │      │ CPU限额: 20%         │   │
│  │ 内存: 8GB       │      │ 内存: 2GB            │   │
│  │ 网络: 直连券商   │      │ 网络: Cloudflare    │   │
│  │                 │      │         Tunnel     │   │
│  └─────────────────┘      └─────────────────────┘   │
│         ↓                          ↓                  │
│    券商 API（TCP）           Cloudflare 边缘           │
└──────────────────────────────────────────────────────┘
```

**资源分配（`docker-compose.yml`）**：

```yaml
services:
  quant:
    image: hengcang/quant:latest
    restart: unless-stopped
    cpus: "5.0"              # 最多吃 5 个 CPU 核
    mem_limit: 8g
    mem_reservation: 4g       # 保底 4GB
    cpu_shares: 2048          # 相对权重 2 倍（资源紧张时优先）
    network_mode: host        # 量化对延迟敏感，host 网络省 NAT
    env_file: .env.quant      # 券商密钥单独文件
    volumes:
      - ./data/quant:/app/data
      - ./logs/quant:/app/logs
    healthcheck:
      test: ["CMD", "python", "healthcheck.py"]
      interval: 30s
      timeout: 5s
      retries: 3

  hengcang:
    image: hengcang/backend:latest
    restart: unless-stopped
    cpus: "2.0"               # 最多吃 2 核
    mem_limit: 2g
    cpu_shares: 1024          # 相对权重 1 倍（劣后）
    ports:
      - "127.0.0.1:8000:8000"   # 只监听本地，靠 Tunnel 暴露
    env_file: .env.hengcang
    volumes:
      - ./data/hengcang:/app/data
    depends_on:
      - quant                   # 衡仓启动不能拖慢量化

  tunnel:
    image: cloudflare/cloudflared:latest
    restart: unless-stopped
    command: tunnel --no-autoupdate run --token ${CF_TUNNEL_TOKEN}
    cpus: "0.5"
    mem_limit: 256m
```

**注意事项**：

- ✅ 量化用 `network_mode: host`，省掉 Docker NAT 带来的 ~0.3ms 延迟
- ✅ `cpu_shares` 让 CPU 紧张时量化优先
- ✅ 量化密钥 `.env.quant` 与衡仓密钥 `.env.hengcang` **分文件**，权限 600
- ✅ 衡仓端口只绑 `127.0.0.1`，绝不暴露公网，外部走 Tunnel
- ❌ **不要**把量化接口也通过 Tunnel 暴露——任何公网可达就是攻击面
- ⚠️ **单机风险**：衡仓 OCR 吃满 CPU 可能影响量化。通过 `cpus` 限额 + `cpu_shares` 权重缓解，但不能 100% 避免

### 方案 B：双机分离（推荐）

**架构**：

```
[量化专用机]                           [衡仓机]
i5+ / 16GB / 有线网                    旧笔记本 / 小主机
  ↓                                     ↓
券商 TCP 长连接                         Cloudflare Tunnel
  ↓                                     ↓
（独立 UPS 供电）                       （随便，停电无所谓）
```

**硬件建议**：

| 角色 | 推荐配置 | 成本 |
|------|---------|------|
| 量化专用机 | Intel NUC 13 i5 / 16GB / 512GB NVMe | 3000–4500 元 |
| 衡仓机 | 淘汰的笔记本 / 树莓派 5 | 0–800 元 |
| 量化机 UPS | APC BE650G 或山特 TG-BOX 850 | 400–600 元 |
| 网络 | 千兆光纤主线 + 4G/5G 备用（USB 网卡 + 手机热点） | 月费 10–30 元 |

**双机优点**：

- ✅ 衡仓崩了不影响量化下单，量化崩了不影响衡仓访问
- ✅ 衡仓可以随便重启做更新（量化交易时段严禁重启）
- ✅ 衡仓机可以晚上关机省电（量化机 24/7）
- ✅ 安全面分离，衡仓被攻击时量化账户不暴露

**布线建议**：两台机都接路由器，量化机用**有线**网口（Wi-Fi 延迟抖动大），路由器 QoS 里给量化机 IP 最高优先级。

### 方案 C：券商托管 VPS

**原理**：部分券商（华泰 / 国泰君安 / 中信 / 东方财富等）提供**金融云 VPS**，机房离上交所 / 深交所机房最近，延迟 < 1ms。

**价格**（2026 年大致行情）：

- 入门：1 核 2G，100–200 元/月
- 常规：2 核 4G，300–500 元/月
- 企业：4 核 8G+，1000 元+/月

**门槛**：需要是该券商的客户，部分要求资产 / 交易量达标才能开通。

**适用场景**：

- 做**日内高频 / 套利 / Tick 级**策略（毫秒延迟对盈亏有实际影响）
- 你经常出差 / 在外地（家里电脑不稳定）
- 策略本金大（比如 100 万+），VPS 月费相对收益忽略不计

**不适用**：

- 日频 / 周频策略（家里 50ms 延迟没区别）
- 本金小于 10 万（VPS 月费占比太高）

### 网络与可靠性（量化必做）

**双线冗余**：

- 主线：电信 / 联通光纤（稳定性 > 移动）
- 备线：4G / 5G USB 网卡，用 **mwan3**（OpenWrt 路由）或 Windows 上的 `NetSetMan` 做自动切换
- 或更简单：手机开热点，主线断时手动切

**UPS + 路由器挂 UPS**：

- 停电瞬间能让量化进程**优雅平仓 / 撤单**（写 shutdown 钩子）
- 典型断电 = 策略错仓一整天，UPS 的 400 元回本很快

**监控与告警**：

- **本地**：Prometheus + Grafana（免费）监控量化进程 CPU / 内存 / 连接状态
- **远程**：用 `uptime-kuma` 或 `healthchecks.io` 每分钟拉一次量化 `/health`，连续 2 次失败推送到**微信**（Server 酱 / PushPlus 免费）
- **关键告警**：
  - 量化进程挂了 → 立即推送
  - 券商连接断开 → 立即推送
  - 持仓与预期不一致 → 立即推送（防乌龙单）
  - 单日亏损超阈值 → 立即推送 + 自动停策略

```python
# quant/risk_guard.py 片段示例
MAX_DAILY_LOSS_PCT = float(os.getenv("MAX_DAILY_LOSS_PCT", "2.0"))

def check_risk(account_equity, day_start_equity):
    loss_pct = (day_start_equity - account_equity) / day_start_equity * 100
    if loss_pct > MAX_DAILY_LOSS_PCT:
        emergency_stop_all_strategies()
        push_wechat(f"🚨 日亏损超 {MAX_DAILY_LOSS_PCT}%，已自动停止策略")
```

### 密钥与安全边界

**严格分级**（从最敏感到最低）：

| 等级 | 内容 | 存放 |
|------|------|------|
| 🔴 顶级 | 券商账号密码、资金密码、U盾口令 | 专用机的环境变量 + 系统钥匙串，**绝不**写入任何仓库 |
| 🟠 高 | 券商 API Token、下单签名密钥 | `.env.quant`，权限 600，不同步到 git |
| 🟡 中 | Anthropic API Key、行情数据订阅密钥 | `.env.hengcang`，权限 600 |
| 🟢 低 | Cloudflare Tunnel Token | 环境变量即可 |

**网络隔离**：

- 量化机设置**出站白名单**防火墙，只允许访问：
  - 券商交易 IP（固定几个）
  - 行情源（同花顺 L2 / 米筐 / tushare 等）
  - NTP 时间同步
  - 告警推送接口（微信 / Server 酱）
- **禁止**出站到：社交网站、广告网络、任何无关域名
- 用 Windows 防火墙 / iptables / pfSense 实现

**两台机分 VLAN**：家里路由器若支持 VLAN，把量化机单独一个段，衡仓机另一个段，互相不能访问。进一步防"衡仓被打穿 → 量化账户泄漏"。

### 市场时段与自动化

**作息策略**：

| 时段 | 衡仓 | 量化 |
|------|------|------|
| 夜间 / 周末 | 正常运行 | 可重启、更新、跑回测 |
| 盘前（9:00–9:25） | 正常运行 | 启动策略、同步持仓、心跳检测 |
| 盘中（9:30–11:30 / 13:00–15:00） | 正常运行 | **冻结变更**，严禁重启 / 发版 |
| 收盘后（15:00–16:00） | 正常运行 | 对账、日报、落盘 |

**自动化脚本**：

```bash
# cron 条目示例
# 早 8:50 预热量化（加载模型 / 检查连接）
50 8 * * 1-5 docker exec quant python warmup.py

# 晚 15:30 收盘对账
30 15 * * 1-5 docker exec quant python reconcile.py

# 周末 02:00 自动更新量化（避开交易时段）
0 2 * * 6 docker-compose pull quant && docker-compose up -d quant
```

### 量化并行最小 Checklist

开写量化代码前，先把这些基础设施做好：

- [ ] 量化机独立于衡仓机（方案 B），或至少 Docker 隔离（方案 A）
- [ ] 量化机装 UPS，停电能撑 15 分钟以上
- [ ] 券商密钥存 **环境变量 + 系统钥匙串**，`.env.quant` 权限 600，不入仓库
- [ ] 量化代码里有**风控兜底**（日亏损熔断 / 单笔限额 / 持仓白名单）
- [ ] `/health` 端点 + 外部 uptime 监控 + 微信告警通道打通
- [ ] 策略代码变更走**夜间 / 周末发版**，交易时段严禁动
- [ ] 所有下单记录 append-only 写本地日志（防篡改，便于事后审计）
- [ ] 有**手动应急平仓脚本**，量化崩了能一键清仓
- [ ] 家里宽带 + 4G 备线，路由器 QoS 给量化机最高优先级
- [ ] 量化出站流量白名单（只允许券商 IP + 行情源 + 告警通道）
