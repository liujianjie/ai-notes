# 衡仓（Hengcang）— 产品改名与域名方案

> 原项目 `F:\AIProject\StockTrading` / 资产配置追踪器，已于 2026-05-03 敲定对外产品名为 **"衡仓"（Hengcang）**。本文档记录命名决策与域名落地方案。

## 一、产品名：衡仓（Hengcang）

### 为什么改名

现仓库名 `StockTradingAnalysis` 误导为"交易执行 / 选股推荐"工具，但项目实际价值是**个人资产配置 + 再平衡追踪**——多路径导入持仓、目标配比对照、AI 配比建议、多端（H5 / 微信小程序 / Android）。

### 为什么选"衡仓"

| 维度 | 说明 |
|------|------|
| **核心卖点对应** | "衡" = 再平衡（rebalance，仪表盘目标配比 vs 实际占比的核心动作）；"仓" = 持仓 / 仓位 |
| **专业感** | 面向股民用户直观，不像"配比"偏日常词 |
| **搜索友好** | 拼音 `hengcang` 冷门，百度 / 微信搜索零污染，长期 SEO 干净 |
| **兼容未来路线** | AI 功能可叫"衡仓 AI 配比建议"；多用户版本可叫"我的衡仓"；多端统一品牌 |
| **域名可得性** | `.com` 大概率能直接拿下（无需抢注 / 溢价收购） |

### 备选名（已否决，留档）

| 候选 | 否决原因 |
|------|---------|
| 配比 / Peibi | 太日常，专业感不足 |
| 配衡 / Peiheng | 拼音冷门，但"衡仓"更聚焦"仓位管理" |
| 投衡 / Touheng | 偏"投资"，弱于"配置 + 平衡"的组合 |
| 智配 / Zhipei | "智 X"系品牌泛滥，辨识度低 |
| Rebal / Allo | 英文品牌对中文用户搜索不友好 |

## 二、域名方案

### 候选清单（按推荐优先级）

| 排序 | 域名 | 价格（Cloudflare 批发） | 备注 |
|------|------|----------------------|------|
| ⭐ 1 | **`hengcang.com`** | ~$10.44 / 年 | 冷门拼音 + `.com`，可注册概率**很高**。首选目标 |
| 2 | `hengcang.app` | ~$14 / 年 | Google 强制 HSTS，天然 HTTPS，配 uniapp / PWA 语义精准 |
| 3 | `hengcang.io` | ~$32 / 年 | 科技感最强，但价格贵 2–3 倍 |
| 4 | `myhengcang.com` | ~$10 / 年 | 备选 1：若 `hengcang.com` 意外被占 |
| 5 | `hengcangapp.com` | ~$10 / 年 | 备选 2 |

### 避坑

- ❌ `.xyz` / `.top` — 邮件发送 / 外链权重被降级，验证邮件易进垃圾箱
- ❌ `.cn` — 需要 ICP 备案 + 身份验证，个人项目太麻烦
- ❌ 连字符（`heng-cang.com`）— 口播没法念
- ❌ 数字 homophone（`hengcang8.com`）— 看起来像灰产

## 三、执行清单

### 第 1 步：注册域名

1. 打开 [dash.cloudflare.com](https://dash.cloudflare.com) → 左侧 **Domain Registration → Register Domains**
2. 搜索 `hengcang.com`
3. 如果显示 "Available"，直接下单（账单周期建议 1 年起步，续费无加价）
4. 如果被占，按上面排序依次搜索 2–5

### 第 2 步：安全加固

下单完成后立刻：

- ✅ 账户开启 **2FA**（Registrar 被劫持 = 域名被盗）
- ✅ Domain Registration → Manage → **Transfer Lock: On**（防止未授权转出）
- ✅ **SSL/TLS → Edge Certificates → Always Use HTTPS: On**
- ✅ **SSL/TLS → Edge Certificates → Automatic HTTPS Rewrites: On**

### 第 3 步：DNS 绑定（按托管平台三选一）

#### A. Cloudflare Pages（最省事）

Pages 项目 → **Custom domains → Set up a custom domain** 输入域名即可，DNS 自动写入，SSL 自动签发。

#### B. Vercel

Vercel 项目 → Settings → Domains 添加域名，然后在 Cloudflare DNS 添加：

```
类型    名称    内容                          代理状态
A       @      76.76.21.21                   DNS only (灰云)
CNAME   www    cname.vercel-dns.com          DNS only (灰云)
```

> ⚠️ 代理状态必须设为 **DNS only（灰云）**，否则 Vercel 的 SSL 握手会失败。

#### C. Netlify

```
类型    名称    内容                                  代理状态
CNAME   @      <你的站点>.netlify.app                DNS only
CNAME   www    <你的站点>.netlify.app                DNS only
```

## 四、项目内改名改动范围

改名成本不高，需要改的露出点：

| 文件 / 位置 | 改动 |
|------------|------|
| `F:\AIProject\StockTrading\README.md` 第 1 行 | `# StockTrading — 资产配置追踪器` → `# 衡仓 Hengcang — 个人资产配置与再平衡追踪器` |
| `F:\AIProject\StockTrading\CLAUDE.md` | 项目介绍段落 |
| `frontend/package.json` / `frontend-uniapp/package.json` | `name` 字段 |
| 页面 `<title>` 标签（`frontend/index.html` 等） | "衡仓 \| 资产配比追踪" |
| `backend/` FastAPI `app = FastAPI(title=...)` | "衡仓 API" |
| `frontend-uniapp/manifest.json` | uniapp 应用名 / 小程序名称 |

**不用改**：代码里的类名 / 变量名（`Holding`、`Portfolio` 等都是通用领域词，和产品名无关）。

**不用改**：GitHub 仓库名 `liujianjie/StockTradingAnalysis`（产品名与仓库名解耦，改仓库名会影响 clone URL / 外链）。
